# LOF溢价率套利工具 - AI开发Prompt

---

## 一、项目概述

```
【项目名称】LOF溢价率套利计算器

【项目背景】
开发一个纯前端工具，用于实时计算LOF（上市开放式基金）的溢价率，
辅助判断套利机会。项目纯自用，无需上架任何应用商店。

【使用场景】
- 设备：iPhone手机 + PC电脑
- 环境：需要联网获取实时数据
- 频率：交易日盘中使用，每30秒刷新一次数据
```

---

## 二、技术架构要求

```
【架构原则】
- 纯前端为主，Serverless Function为辅
- 零服务器成本，无需域名备案
- 苹果手机通过Safari添加到主屏幕使用（PWA）

【技术栈】
前端框架：Vue 3 + Vite
UI样式：TailwindCSS 或 响应式CSS
部署平台：Vercel（免费）
API代理：Vercel Serverless Functions（/api目录）

【项目结构规范】
├── src/                    # 前端源码
├── api/                    # Serverless Functions（Vercel自动识别）
├── public/
│   ├── manifest.json       # PWA配置
│   ├── sw.js               # Service Worker
│   └── icons/              # PWA图标
└── index.html
```

## 三、功能需求

### 3.1 核心功能

```
【F1 - LOF溢价率计算】
公式：溢价率 = (场内价格 - 基金净值) / 基金净值 × 100%

输入：LOF基金代码、对应市场代码
输出：场内价格、估算净值、溢价率、是否可套利

【F2 - 数据展示列表】
展示所有LOF基金的：
- 基金代码、基金名称
- 场内实时价格
- 估算净值（IOPV）
- 溢价率（带颜色标识：红色=溢价，绿色=折价）
- 支持按溢价率排序

【F3 - 套利收益计算】
考虑因素：
- 申购/赎回费率（默认0.15%）
- 交易佣金（默认万3）
- 时间成本（T+2到账）
- 输出：预期套利收益、年化收益率

【F4 - 自定义关注列表】
- 用户可添加/删除关注的LOF基金
- 数据存储在浏览器localStorage
- 支持导入/导出配置

【F5 - 自动刷新机制】
- 盘中（9:30-15:00）每30秒自动刷新
- 盘后显示收盘数据，不刷新
- 支持手动刷新按钮
- 显示最后更新时间
```

### 3.2 界面要求

```
【UI规范】
- 响应式布局，适配手机和电脑
- 手机端优先设计
- 溢价率阈值高亮：
  - 溢价 > 1%：深红色背景
  - 溢价 > 0.5%：浅红色背景
  - 折价 > 1%：深绿色背景
  - 折价 > 0.5%：浅绿色背景

【PWA要求】
- 支持添加到iOS主屏幕
- 全屏模式运行
- 离线时显示缓存的页面框架
- 自定义应用图标和启动画面
```

## 四、数据源方案

```
【数据获取策略】
通过Vercel Serverless Function作为API代理，
解决浏览器跨域问题，同时保持零服务器成本。

【数据源接口】

1. LOF场内实时行情（东方财富）
   接口：https://push2.eastmoney.com/api/qt/clist/get
   参数示例：fs=b:MK0021&fields=f2,f3,f12,f14
   说明：f2=最新价, f3=涨跌幅, f12=代码, f14=名称

2. 基金实时净值估算（天天基金）
   接口：https://fundgz.1234567.com.cn/js/{基金代码}.js
   返回：JSONP格式，包含gsz（估算净值）

3. LOF基金列表（备用）
   接口：https://fund.eastmoney.com/data/FundGuideapi.aspx
   说明：获取完整LOF基金代码列表

【API代理实现要求】
路径：/api/fund.js, /api/price.js
功能：
- 接收前端请求
- 转发到上述数据源
- 设置CORS头
- 返回JSON数据
- 添加基础错误处理和超时机制
```

## 五、开发注意事项

### 5.1 iOS兼容性

```
【必须遵守】
1. PWA在iOS上的限制：
   - Service Worker功能受限
   - 不支持后台同步
   - 不支持推送通知
 
2. manifest.json配置：
   - display: "standalone"
   - apple-mobile-web-app-capable: yes
   - 需单独配置apple-touch-icon

3. Safari特殊处理：
   - 100vh问题（需用CSS变量修复）
   - 弹性滚动问题
   - 安全区域适配（刘海屏）
```

### 5.2 性能要求

```
【性能指标】
- 首屏加载：< 2秒
- API响应：< 500ms
- 列表渲染：支持100+基金流畅滚动
- 内存占用：< 50MB

【优化策略】
- 静态资源CDN（Vercel自带）
- API响应缓存（30秒内复用）
- 虚拟列表（如果基金数量多）
- 图片懒加载（如果有）
```

### 5.3 错误处理

```
【必须处理的异常】
1. 网络请求失败 → 显示重试按钮
2. 数据解析失败 → 跳过该基金，显示警告
3. API限流 → 降低请求频率
4. 非交易时间 → 显示提示信息
5. 数据异常值 → 过滤或标记
```
